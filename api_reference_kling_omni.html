Kling Omni API Reference

Python examples and parameter reference for Kling's Omni video model (O1 and v3). Based on Kling Omni docs and useapi.net. No SDK—use pip install requests.

Workflows

Omni supports multiple workflows. The workflow is chosen by which inputs you provide:

InputsWorkflow
frame_start (+ optional frame_end)Frames — first and last frame interpolation
video_1 + video_mode=transformVideo Transform — modify a reference video
video_1 + video_mode=reference (default)Video Reference — style guided by video
Images/elements only, or text-onlyDefault — text-to-video with optional refs

1. Default workflow (text + image reference)

import requests

API_URL = "https://api.useapi.net/v1/kling/videos/omni"  # or Kling official API
headers = {
    "Authorization": "Bearer YOUR_API_TOKEN",
    "Content-Type": "application/json",
}

# Reference images in prompt with @image_1, @image_2, etc.
response = requests.post(API_URL, headers=headers, json={
    "prompt": "A woman @image_1 dancing elegantly in a ballroom",
    "duration": "5",
    "aspect_ratio": "16:9",
    "image_1": "https://example.com/person.jpg",
})

task = response.json()
task_id = task["task"]["id"]
print(f"Task: {task_id}")
# Poll GET /tasks/{task_id} for status

2. Frames workflow (first + last frame)

# Generate video between start and end frame images
# Aspect ratio is auto-derived from input when using frames
response = requests.post(API_URL, headers=headers, json={
    "prompt": "Smooth cinematic transition from start to end",
    "duration": "5",
    "frame_start": "https://example.com/start_frame.jpg",
    "frame_end": "https://example.com/end_frame.jpg",  # optional
})

# When using frames: image_N, element_N, video_1 are NOT allowed
# aspect_ratio is always "auto" for frames workflow

3. Frames workflow (start frame only)

# Animate from a single start frame
response = requests.post(API_URL, headers=headers, json={
    "prompt": "Scene comes to life with gentle movement",
    "duration": "5",
    "frame_start": "https://example.com/first_frame.jpg",
})

4. Video reference workflow

# Use reference video to guide style/motion
# Duration is locked to video length (3-10s O1, 3-15s v3)
response = requests.post(API_URL, headers=headers, json={
    "prompt": "Scene like @video_1 but in a winter setting",
    "video_1": "https://example.com/reference.mp4",
    "video_mode": "reference",  # default
    "keep_audio": False,        # optional
})

5. Video transform workflow

# Directly transform/modify the reference video
response = requests.post(API_URL, headers=headers, json={
    "prompt": "Make it look like a vintage film",
    "video_1": "https://example.com/source.mp4",
    "video_mode": "transform",
})

6. Multi-shot (v3 only)

# Split into multiple shots with separate prompts
# Min 2 shots, total duration 3-15 seconds
response = requests.post(API_URL, headers=headers, json={
    "omni_version": "v3",
    "shot_1_prompt": "A cat walks into a room",
    "shot_1_duration": "3",
    "shot_2_prompt": "The cat jumps onto the couch",
    "shot_2_duration": "4",
})
# Cannot use prompt or duration with multi-shot

Parameter Reference

omni_version

ValueDescription
v3Default. Longer duration, multi-shot, video elements.
o1Shorter range, no multi-shot.

mode

ValueDescription
stdDefault. Standard generation.
proProfessional mode (higher quality).

duration
